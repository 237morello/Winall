# Mission

Tu es un Software Architect Senior spécialisé en Analytics, Next.js App Router, PostgreSQL, Prisma, Recharts et Microsoft Clarity.

Nous voulons construire un système nommé "Winall Analytics" inspiré de Microsoft Clarity mais plus léger.

L'objectif n'est PAS de remplacer Clarity.

L'objectif est de fournir dans notre interface Admin les 50% d'informations les plus utiles afin que les administrateurs puissent surveiller rapidement le comportement des utilisateurs sans devoir ouvrir Clarity.

Pour les analyses avancées (heatmaps, session replay, rage clicks, dead clicks, scroll maps, etc.), nous continuerons d'utiliser Microsoft Clarity.

# Contexte technique

Stack :

* Next.js App Router
* TypeScript
* Prisma
* PostgreSQL
* Shadcn UI
* Tailwind CSS
* Recharts
* Microsoft Clarity

# Contraintes

1. Architecture scalable.
2. Faible impact sur les performances.
3. Compatible SSR et App Router.
4. Compatible utilisateurs connectés et visiteurs anonymes.
5. Respect RGPD.
6. Aucune donnée sensible.
7. Possibilité d'ajouter de nouvelles métriques facilement.

# Utilisation de Microsoft Clarity

Nous utiliserons :

window.clarity("event", ...)
window.clarity("set", ...)
window.clarity("identify", ...)

afin de relier nos événements métiers avec les données Clarity. Microsoft Clarity permet également les événements personnalisés et les tags personnalisés. Les événements et tags devront être intégrés à notre architecture analytics.

Nous souhaitons également étudier l'utilisation future du Data Export API de Clarity pour enrichir nos tableaux de bord internes.
